Icon Retrieval Evaluation — Embedding Files
============================================

Each .npy file is a float32 NumPy array with rows aligned to the 100 evaluation
icons in data/eval_data.json (in the same order they appear under "icons").

To load:
    import numpy as np
    mat = np.load('eval_text_embeddings.npy')   # shape (100, 3072)

Files
-----

eval_text_embeddings.npy   (100, 3072)  float32
    Gemini gemini-embedding-001 embeddings of each icon's text description.
    Embedded with task_type=RETRIEVAL_DOCUMENT.
    These are the icon-side vectors for the plain_embed and qexp_plain search methods.

eval_hyde_from_img.npy     (100, 3072)  float32
    Gemini gemini-embedding-001 embeddings of hypothetical search queries
    generated by Gemini Vision from each icon image ("what would someone search
    to find this icon?"). Embedded with task_type=RETRIEVAL_DOCUMENT.
    Icon-side vectors for hyde_from_img, qexp_hyde_img, and hyde_query_hyde_img methods.

eval_hyde_from_prompt.npy  (100, 3072)  float32
    Gemini gemini-embedding-001 embeddings of hypothetical search queries
    generated by Gemini from each icon's text description.
    Embedded with task_type=RETRIEVAL_DOCUMENT.
    Icon-side vectors for hyde_from_prompt, qexp_hyde_prompt, hyde_query_hyde_prompt methods.

eval_caption_embeddings.npy  (100, 3072)  float32
    Gemini gemini-embedding-001 embeddings of Gemini Vision long captions for
    each icon image. Embedded with task_type=RETRIEVAL_DOCUMENT.
    Icon-side vectors for caption_embed, qexp_caption, hyde_query_caption methods.

eval_image_embeddings.npy  (100, 768)   float32
    SigLIP2 (google/siglip2-base-patch16-224) image embeddings for each icon.
    L2-normalized. Used for the siglip2 and siglip2_caption search methods.

Note: All vectors are L2-normalized (unit norm), so cosine similarity reduces to
a dot product: scores = embeddings @ query_vec
