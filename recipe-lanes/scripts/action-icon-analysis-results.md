## Key Findings

**A fixed exact-match icon library is not viable for action nodes.**

- 2449 unique descriptions out of 2961 total action nodes = **82.7% uniqueness ratio**. Nearly every action node has a description that appears only once.
- Top 50 exact matches cover only **8.3%** of action nodes. Top 20 covers 4.4%. The distribution is extremely flat — there is no long tail to exploit.
- You need **969 unique descriptions** to cover just 50% of action nodes; 2153 for 90%. Compare this to ingredients, where a small fixed set covers most cases.
- **Root cause:** Action `visualDescription` values are generated as verbose, context-specific visual scenes ("Whisked Eggs Poured Into A Non-stick Pan With Melted Butter, Cooking And Gently Stirring") rather than canonical action verbs. These are nearly always unique.
- **The semantic clustering shows the real structure:** The same 2961 action nodes collapse into ~11 visual archetypes (mix/stir, chop/cut, fry/sauté, boil/simmer, bake/roast, steam/heat, add/season, rest/cool, press/shape, drain/strain, serve/plate). A keyword-based matcher on the description text would yield dramatically higher coverage than exact-string matching.
- **Recommended approach:** Instead of a fixed library, normalize action icon generation at the *prompt level* — instruct the model to use short canonical action labels ("whisking eggs in bowl", "pan with hot oil") rather than scene descriptions. This would make the existing cache hit-rate mechanism effective for action nodes, the same way it already works well for ingredients (67.8% hit rate vs 11.6% for actions currently).
- **Alternative:** A keyword → canonical-icon mapper applied at lookup time would also work without changing the prompt, but requires maintaining the keyword rules.
