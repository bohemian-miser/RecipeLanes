# Product & Architecture Summary: RecipeLanes

## 1. The Core Concept (The "Why")

**Shift from "Linear Text" to "Topological Time-Flow"**

Traditional recipes are linear lists of instructions that poorly convey the concurrency inherent in cooking. A cook reading "Step 4: Whisk egg whites" while "Step 1: Onions caramelizing" is still happening risks burning the onions. 

**RecipeLanes** reimagines recipes as **Time-Flow Graphs**:
*   **X-Axis (Lanes):** Represents *resources* or *contexts* (e.g., "Prep Station", "Stove - Burner 1", "Oven", "Stand Mixer"). This physically separates tasks by location/tool.
*   **Y-Axis (Time):** Time flows downwards. A node placed lower than another implies it happens later.
*   **Topology:** Dependencies (arrows) dictate the critical path. A step like "Combine wet and dry ingredients" is a *merge node* that cannot start until both the "Wet Mix" thread and "Dry Mix" thread are complete.

This visualization immediately highlights bottlenecks, parallel opportunities (multitasking), and resource contention, making it the "Gantt Chart for Chefs."

## 2. The User Experience (The "GitHub for Cooking" Model)

We adopt a developer-centric mental model applied to culinary arts.

*   **Home Cook (Free Tier):**
    *   **Parser & Visualizer:** Paste any structured recipe text to generate a graph instantly.
    *   **Local Storage:** "My Cookbook" lives in browser local storage.
    *   **Read-Only Access:** Browse public community recipes.
    *   **Scaling:** Automatic ingredient scaling that updates the visual graph metadata.

*   **Creator/Pro (Paid Tier):**
    *   **Private Repositories:** Keep secret family recipes or commercial prep lists private.
    *   **Branding:** Custom fonts, color themes, and logo watermarks for exported SVGs/PDFs (great for food bloggers/YouTubers).
    *   **High-Res Export:** 4k PNG/SVG export for print.

*   **Community:**
    *   **Forking:** "Fork" a recipe (e.g., "Kenji's Bolognese") to your own account to tweak it (e.g., "Add Miso").
    *   **Remixing:** Merge sub-graphs (e.g., import a "Perfect Pie Crust" graph into a "Apple Pie" recipe).
    *   **Pull Requests (Future):** Suggest improvements to the original author.

## 3. The AI & Data Pipeline

The system bridges the gap between natural language and structured data.

*   **NLP (LLM Integration):**
    *   **Input:** Raw text (URL or paste).
    *   **Process:** An LLM (Gemini/GPT) processes the text into a strict intermediate JSON schema.
    *   **Output:** The schema defines `Nodes` (actions), `Edges` (dependencies), `Resources` (lanes), and `TimeEstimates`.
    *   *Correction UI:* Users can manually drag-and-drop nodes to correct the AI's inference before saving.

*   **Hybrid Icon System ("Cache-First"):**
    *   **Problem:** Recipes need visual cues (icons for "onion", "whisk", "oven"), but a static library is never enough.
    *   **Strategy:**
        1.  **Static Repo:** Check local SVG library for common matches (e.g., `onion.svg`).
        2.  **Database Lookup:** Check Supabase `icons` table for previously generated assets by other users.
        3.  **AI Generation (Fallback):** If missing, call an Image Generation Model to create a simple, vector-style line icon.
        4.  **Cache:** Save the new icon to Supabase for all future users.

## 4. The Technical Stack

*   **Frontend:**
    *   **Framework:** **Next.js (React + TypeScript)**. Provides server-side rendering for SEO (public recipes) and fast client-side interactivity.
    *   **State:** React Context / Zustand for managing the live graph state.
    *   **Visualization:** SVG-based rendering (custom or D3.js) for the interactive swimlane graph.

*   **Backend:**
    *   **API:** **Next.js Serverless API Routes**. Acts as a secure proxy to external services (LLM APIs, Image Gen) to hide API keys.
    *   **Auth:** Supabase Auth (Magic Links / Social Login).

*   **Database & Storage:**
    *   **Database:** **Supabase (PostgreSQL)**. Stores User profiles, Recipe JSON blobs (structured data), and metadata (fork relationships).
    *   **Storage:** **Supabase Storage**. Hosts user avatars and the growing library of AI-generated icon assets.
